GIOTTO DYNAMIC ACCESS CONTROL DOCUMENTATION

This is the guide to use the Dynamic Access Control Android Application. We have built two Android Applications, which demonstrate the level of Access Control in Building Depot. Here we use two contexts:

  1. Role of the client (ADMIN / NON-ADMIN)

  2. Location of the client (vicinity to IoT space)

The two applications are:

  1. GIOTTO

    2.    GIOTTO uses static locations, ie. we can emulate locations in the App and check for sensor data in that location. For eg, if we have deployed sensors that are talking to BuildingDepot in a space that we call Google, in the app there is a dropdown that holds a list of locations. On selecting Google, we are able to view the sensors deployed in that location. So, to use this App it is not necessary that we are to be physically present in that location.







  2. BLE_GOOGLETEST

    3.    The BLE_GOOGLETEST app, serves the same purpose as GIOTTO app. The only difference plays in physical location of the user. This app requires that the user be physically present at the location where the sensors have been deployed.





To use these applications, you should have installed and properly configured Building Depot
