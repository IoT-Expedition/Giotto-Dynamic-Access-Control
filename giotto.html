Giotto Application Documentaion

The working of the GIOTTO apk is explained in this documentation. The working is explained in steps as below:

  1. All the necessary information like OAuth keys, user type's and email's are supplied through the config file. Also, each HTTP/HTTPS request to the BuildingDepot Server requires an OAuth Authorization header. Hence we generate an access_token using the OAuth client credentials supplied. We generate access_tokens in the config.java file, in the getToken() function

  2. The MainActivity activity does the basic credential check. In BuildingDepot, the Location Sensor holds the user's credentials as METADATA. From the app, th credentials suppplied in the config file are verified with the metadata of user Location Sensor. The user could be Admin/ Non-Admin. Once authorized, it moves to the next page.

  3. In the Devices Activity,the Location Dropdown is populated with the location provided

Configuring the APK’s:

  1. Download the Giotto and Ble_Google_tese APK’s from: https://github.com/IoT-Expedition/Giotto-Dynamic-Access-Control
  2. Open them on Android Studio or Eclipse. Preferably Android Studio
  3. Navigate to config.java file
  4. Edit these fields:
    1. ip  - Give the URL on wich you have BuildingDepot installed (Please also specify the PORT of dataservice)
    2. location  - Enter a location your sensors are in
    3. userType – On one phone, specify as ADMIN and NON-ADMIN on the other phone
    4. beacon_mac – Give your BLE beacon’s MAC ID



  5. Download both the APK’s after the edits into both the phones, with one as ADMIN and the other as NON-ADMIN
